# RimLife

- 关于 RimLife：基于 LLM 的游戏剧情设计与增强模组
- 定位：
-- 通过预处理底层数据、结构化剧情模板，降低需求解读阶段对 LLM 注意力的消耗，提高模型生成质量的下限
-- 对于能力更强的模型，则有助于降低输入token量，节省成本

## 原理

- 数据准备
-- 系统根据游戏内事件、数据和玩家行为自动挑选合适的剧情剪辑之主旨(剧本卡 ScriptCard)
-- 剧本卡需要提前撰写，其中预留可动态填充的叙事模板(Template)
-- 剧本中另外有预设标签和参数，用于设定事件卡(EventCard)中对叙事元素(Narrative)的匹配规则
-- 系统围绕角色实时维护可写事件卡和事件池(EventPool)
-- 底层游戏数据将转化为语素(Lexeme)，作为事件卡中叙事元素的候选项

- 剧情生成
-- 首先确定参与会话的角色，再根据实际情况挑选一个剧本卡
-- 系统逐个检测并填充剧本卡中的模板，其中标签和参数作为匹配规则挑选对应的角色和对应的事件池
-- 填充时，逐个检查事件池中的事件卡，找到最合适的叙事元素，再填充其语素文本
-- 这个过程生成自然语言描述的剧情梗概，用于构造 LLM 提示词
